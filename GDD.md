## 一、游戏概述

### 1.0 背景故事

人类在 R-32 行星定居 200 年后，遭遇未知外星怪物的突袭。殖民地核心反应炉是唯一能源来源。你是指挥防卫舰的指挥官，操控飞船在战场上空部署防御塔和无人机制造核心，保护反应炉不被摧毁。

### 1.1 一句话描述
玩家操控可移动防御飞船，在超大固定路径地图上，于操作范围内建造/升级防御塔，保护地图中心反应炉。

### 1.2 核心信息

| 项目 | 内容 |
|------|------|
| 类型 | 2D 俯视 — 动作策略塔防 |
| 操作 | WASD / 鼠标右键移动飞船 → 自动攻击范围内敌人 → 左键选择、建造与升级塔 |
| 胜负 | 反应炉被摧毁 = 失败；所有敌人被消灭 = 胜利 |
| 背景 | 科幻 — 星际殖民，R-32 行星，外星怪物袭击 |
| 灵感 | 操作: 大鱼吃小鱼；塔防: 保卫萝卜 |
| 平台 | Web (Phaser.js 3) → 后续 PWA |

## 二、游戏模式

### 2.1 关卡模式
- 预设关卡包，固定波次 + 敌人配置
- 通关解锁下一关
- 部分塔在特定关卡锁定/强制携带（新手引导）
- 关卡间数据**不继承**
- 胜利条件: 消灭全部敌人

### 2.2 无尽模式
- 支持选择无尽地图；当前开放 **无尽模式一**
- 无限波次递增
- 追求最长存活/最高分
- 波间休息时间固定
- 每完整撑过 5 波获得 1 枚星核，奖励数量为 `floor(完成波数 ÷ 5)`
- 星核在反应炉被摧毁或玩家主动退出本局时一次性结算并永久保存；不足 5 波不获得星核
- 主动退出只统计已经完整结束的波次，当前仍在进行的波次不计入奖励
- 每局设置一次性结算标记，主动退出、游戏失败和返回主页不得对同一局重复发放星核
- 保存历史最高完成波数

### 2.3 主页面与模式入口
- 主页面提供无尽地图与关卡选择入口，并显示当前持有的星核
- 游戏结束界面显示完成波数、本局星核奖励和持有总数
- 游戏结束后可返回主页面，重新开始无尽模式或进入局外养成；暂停菜单主动退出时先显示完成波数与可得星核，确认后结算并返回主页
- 关卡模式当前开放 5 个关卡：**边境蛇道、双门汇流、环形矿区、裂谷三线、远征试验**
- 关卡模式通关奖励为每关 1 枚星核；失败或主动退出未通关关卡不发放星核
- 局外养成与塔图鉴界面统一命名为 **星核整备舱**

## 三、核心循环

```
[局外配置: 选塔] → [开局]
                        │
  波次开始 ←─────────────────────┘
    │
    ├─→ 敌人沿固定路径涌向中心反应炉
    │     │
    │     ├─→ 飞船移动 + 自动攻击（前期有效）
    │     └─→ 在飞船范围内建造/升级塔（主力输出）
    │
    ├─→ 敌人触碰反应炉 → 持续攻击减 HP
    │
    ├─→ 所有敌人被消灭 → 波次结束
    │
    ├─→ 准备阶段（固定时长，自动进入下一波）
    │     └─→ 飞船自由移动建造 + 跑位
    │
    └─→ 下一波（出生点有方向提示）

核心 tension: 飞船必须在地图上移动才能扩展防线，但地图超大 → 移动是战略资源
```

## 四、玩家飞船

### 4.1 操作
| 行为 | 方式 |
|------|------|
| 移动 | WASD，或鼠标右键点击目标位置自动移动；鼠标左键不控制飞船移动 |
| 攻击 | 导弹锁定（范围 1200，优先高危险→最近敌人），伤害 20（单体），冷却 8s，弹速 800 |
| 建造塔 | 在操作范围（1200）内点击地面 → 选类型 → 引导读条 → 完成 |
| 升级塔 | 在操作范围（1200）内点击塔 → 升级 → 读条 |
| 中断建造/升级 | 按 Esc，或离开 1200 码操作范围、飞船死亡、目标失效时自动中断 → 不扣能量 |
| 出售 | 可卖塔，回收 50% 能量 |
| 退出/暂停菜单 | Esc 依次优先取消读条、退出建造模式或取消塔选择；当前无可退出状态时才进入暂停菜单。空格始终切换暂停；暂停时冻结战斗、物理、波次计时、资源增长、建造/升级读条和动画；菜单可选择继续游戏或主动结算并返回主页面 |

### 4.2 属性

| 属性 | 说明 | 成长 |
|------|------|:--:|
| 操作范围 | 1200（攻击/建造/升级统一） | 不成长 |
| 移动速度 | 500 | 局内 + 局外 |

### 4.3 飞船与敌人交互

- 飞船与敌人碰撞 → **死亡**（敌人不主动锁定飞船）
- 飞船与塔之间无碰撞体积
- 飞船与敌人有碰撞检测，与建筑无碰撞

### 4.4 死亡惩罚
死亡 → 5s 复活倒计时 → 复活在中心反应炉位置 → 损失时间 + 丢失当前位置的战略优势
死亡期间塔继续工作，但无法建造或升级；进行中的读条立即取消且不扣能量。

### 4.5 飞船定位演变

| 阶段 | 作用 |
|------|------|
| 前期 | 主力输出（导弹 20）+ 建塔 |
| 中期 | 输出辅助 + 建造/升级主力 |
| 后期 | 建造与维护，导弹主要用于补刀 |

### 4.6 成长系统

**局内**: 无。飞船属性由局外养成决定。

**局外**: 通关获星核 → 永久提升飞船属性 + 解锁塔类型

## 五、防御塔系统

### 5.1 三种类型

| 类型 | 放置位置 | 危险等级 | 会被攻击？ | 核心机制 |
|------|---------|:------:|:--------:|---------|
| **路径塔** | 仅墙体边缘（必须贴墙） | 0 | 否 | 安全输出，不参与仇恨系统 |
| **阻挡塔** | 任意位置 | 1~3（各塔不同） | 是 | 仇恨半径拦截 + 可被摧毁。危险越高越吸引敌人 |
| **无人机制造核心** | 任意位置 | 0 | 否 | 无碰撞，囤积实体无人机，无人机巡逻+主动迎击 |

**放置规则**: 路径塔只能紧贴墙体边缘放置，不可离墙悬空；选择路径塔时，光标进入墙边 120 码吸附范围后，建造预览与最终落点自动贴合到合法墙边。阻挡塔与无人机制造核心可在任意合法空地自由放置。所有塔均无预设槽位，塔之间不可重叠，无数量上限（能量自然限制）。无人机制造核心无碰撞体积，敌人可穿过，但建造时不能与其他塔或障碍物重叠。

### 5.2 敌人锁定逻辑 — 危险等级系统 v2

**设计理念**：危险等级越高 = 越吸引敌人攻击（负面属性）。危险值由各塔及其等级的属性表独立定义。

#### 核心公式

```
敌人.危险等级 ≤ 实体.危险等级 → 实体可拦截该敌人
```

#### 实体危险等级表

| 实体 | 危险等级 | 可被锁定？ | 规则 |
|------|:------:|:--------:|------|
| 路径塔 | 0 | 否 | 1/2/3 > 0 → 对所有敌人"透明" |
| 无人机制造核心 | 0 | 否 | 1/2/3 > 0 → 对所有敌人"透明"，无碰撞 |
| 阻挡塔 | 见各塔属性/升级表 | 是 | 敌人危险 ≤ 该塔当前危险时可拦截 |
| 主反应炉 | 3 | 是 | 全图兜底候选，对全体敌人生效；被摧毁则失败 |
| 小反应炉 | 3 | 是 | 全图兜底候选，对全体敌人生效；被摧毁只损失产能 |

**反应炉兜底规则**：主反应炉与小反应炉不和阻挡塔竞争危险等级。敌人没有被无人机强制锁定、且当前位置不存在可拦截自己的合格阻挡塔时，才从全图存活反应炉中选择距离最近者作为兜底目标。已经锁定存活反应炉的敌人不会因为同级反应炉出现或距离变化而切换；当前反应炉被摧毁后，才重新选择最近的存活反应炉。

#### 锁定流程 v3（分层兜底）

```
敌人沿路径前往已锁定的反应炉
  │
  ├─→ 进入某塔的攻击范围 → 敌人.危险 ≤ 塔.危险？
  │     ├─→ 否 → 塔透明，无视继续走
  │     └─→ 是 → 加入候选列表
  │
  ├─→ 范围内存在“敌人危险 ≤ 塔危险”的阻挡塔
  │     └─→ 按阻挡塔危险等级降序、同级距离升序锁定
  │     └─→ 已与阻挡塔战斗时保持锁定；仅更高危险阻挡塔可使其切换
  │     └─→ 目标摧毁 → 从当前位置重新扫描
  │
  └─→ 没有合格阻挡塔 → 最近的存活主/小反应炉作为全图兜底目标
```

#### 锁定规则

- **优先级固定**：无人机首伤强制锁定 ＞ 范围内合格阻挡塔 ＞ 最近的存活反应炉
- **危险等级只在阻挡塔层内生效**：危险越高的合格阻挡塔越优先；升级后的危险值按对应塔属性表执行
- **层级切换**：敌人锁定中遇到更高危险的塔 → 切换目标。同级不改
- **目标摧毁后重判**：恢复移动，从当前位置重新扫描候选列表
- **反应炉全图兜底**：主反应炉与小反应炉的仇恨半径均为全图、标称危险均为3，但只在没有合格阻挡塔时参与选敌；未锁定敌人选择距离最近的存活反应炉
- **多敌人可同目标**：多个敌人可以锁定同一座塔，加速其被摧毁
- **锁定中新建更高危险塔**：已锁定的敌人会切换目标到新塔
- **E11反无人机例外**：E11猎翼虫优先锁定自身500码射程内最近的战斗无人机或维修无人机，该优先级高于阻挡塔与反应炉；目标无人机死亡或离开射程后重新扫描。E11不会为了追逐无人机偏离固定路径，射程内没有无人机时才执行普通敌人锁定流程

### 5.3 设计意图

| 机制 | 策略效果 |
|------|---------|
| 塔的仇恨半径 = 攻击范围 | 玩家看到圆圈就知道"这里能拦截"，零学习成本；反应炉固定为全图兜底范围，不受此规则限制 |
| 阻挡塔层内按危险等级降序锁定 | 高危险塔吸引火力，同时不会因反应炉标称危险3而失去拦截作用 |
| 反应炉只作兜底 | 多座反应炉可自然分流未被防线拦截的敌人，且不会让低危险阻挡塔失效 |
| 层级切换 | 敌人遇到更高危险塔会切换 → 肉盾型塔（高危险）保卫输出型塔 |
| 窄通道塔覆盖全宽 | 自然形成堵路效果（无需物理碰撞） |
| 开阔带部分覆盖 | 需要多塔排防线 → 策略布局动机 |

### 5.4 无人机制造核心与无人机

#### 无人机制造核心属性

| 属性 | 说明 |
|------|------|
| 最大驻军 | 各型号不同（5~24 架） |
| 产兵速度 | 各型号不同（5s~20s/架） |
| 核心范围 | 各型号不同（500~700 码） |
| 无人机 HP | 各型号不同（10~50） |
| 伤害/治疗 | 各型号不同 |
| 攻击距离 | 60 码近战，与怪物基础近战距离一致 |
| 追击速度 | 320，不随波次成长 |
| 首次出手 | 锁定目标后至第一发攻击命中前的延迟（与攻击间隔独立） |
| 自回血 | 所有无人机共享：必须实际返回并停靠所属核心后，才以 2HP/s 回复；离开核心时不回血，回血不消耗能量 |

**攻击距离说明**：所有无人机均使用 **60 码近战攻击距离**，与怪物的基础近战距离一致。无人机必须接近目标后才能攻击。

**首次出手速度说明**：无人机迎击敌人 → 进入攻击距离 → 等待"首次出手"时间 → 第一发命中 → 之后按"攻击间隔"循环。例如 D1 首击 0.5s + 攻击间隔 0.7s = 第一发 0.5s、第二发 1.2s、第三发 1.9s。

#### 无人机行为

```
无人机在核心范围内自由巡逻（小幅度随机移动）
  │
  ├─→ D3维修巢无人机：不参与战斗；检测到未满血阻挡塔 → 飞向塔进行维修
  │      └─→ 实际治疗 1HP → 维修无人机自身扣除 1HP；HP归零时死亡
  │
  ├─→ 其他无人机：检测到敌人进入范围 → 主动迎击
  │     └─→ 进入攻击距离 → 持续攻击
  │
  ├─→ 被敌人反击 → HP 归零 → 死亡消失 → 核心重新生产
  │
  └─→ 周围无敌人、无目标、HP 不满 → 飞回所属核心；停靠核心期间自回血（2HP/秒，不消耗能量）
```

#### 全局规则

- 每座无人机核心建成时免费生成首架无人机；之后每次补员均消耗能量：D1 为 2、D2 为 5、D3 为 8 能量/架
- 每座核心独立提供“自动补员耗能：允许/拒绝”开关，默认允许；拒绝或能量不足时保留已完成的生产进度，重新允许且能量足够后立即补员
- 达到当前等级上限时停止补员且不扣能量；升级提高上限后按正常产速与补员费用继续生产

- **首次建造立即产出一架无人机**
- **所有无人机共享回血规则**：无战斗/无目标且HP不满时返回所属核心；只有实际停靠核心期间才以2HP/s回复，离开核心立即停止回血，回血本身不消耗能量
- **所有无人机巢首次建造立即产出一架**
- **核心范围内的全部敌人都可参与索敌分配**；多架无人机会优先分散到当前较少无人机追击的目标
- **进入战斗后保持当前目标**，仅在目标死亡、离开核心范围或失效时重新选敌
- **追击按“是否持续接近目标”判断**：有进展时继续追击，无进展超过1.2s时暂时放弃该目标并重新分配
- **巡逻点只能生成在可通行网格**；连续受墙体阻挡0.65s时清除旧路线并自动恢复巡逻/索敌

#### 无人机对敌人锁定规则的覆盖（最高优先级）

> 敌人被无人机攻击 → **强制切换目标到第一个造成伤害的无人机**。这是锁定规则的唯一例外。

多架无人机同时攻击同一敌人时，**敌人锁定第一个造成伤害的那架**。被锁定的无人机被摧毁后，如果仍有其他无人机在攻击该敌人 → 敌人切到下一个造成伤害的无人机。

```
敌人正在攻击某塔
  │
  └─→ 被无人机攻击
        └─→ 立即切换目标到该无人机 → 击败无人机后恢复移动 → 重新判断候选列表
```

#### 设计意图

| 机制 | 效果 |
|------|------|
| 囤积 | 越早建无人机制造核心，到后期兵越多 → "早投资晚收获"的长线战术 |
| 强制切换 | 无人机吸引火力 → 保护塔和反应炉 → 无人机制造核心的核心价值 |
| 死亡个体不复活 | 被摧毁的无人机个体永久消失；核心可按产速生产新的独立个体 |
| 按产速补充 | 只要敌人清兵速度 < 产兵速度 → 防线越来越厚 |

#### 计算量

每个无人机每帧仅做**距离判断**（对无人机制造核心范围内的敌人），实际计算量：
- 假设 5 座无人机制造核心 × 30 架无人机 × 8 个范围内敌人 ≈ 1,200 次距离判断/帧
- Phaser.js 前端环境完全无压力

### 5.5 塔的攻击目标选择

塔攻击**范围内距离最近的敌人**。每座塔独立判断，无全局优先级。

### 5.6 建造

| 操作 | 流程 |
|------|------|
| 建造 | 飞船在操作范围(1200)内 → 点击自由位置 → 选择类型 → 引导读条 → 完成 |
| 升级 | 飞船在操作范围(1200)内 → 点击升级 → 读条 |
| 取消 | 读条中按取消 → 不扣能量 |
| 出售 | 可卖塔，回收 50% 能量 |

| 项目 | 引导时间 |
|------|:--------:|
| 无人机制造核心 | 1.2s |
| 路径塔 | 1.8s |
| 小反应炉 | 2.0s |
| 阻挡塔 | 2.5s |
| 升至 Lv2 | 1.8s |
| 升至 Lv3 | 2.5s |

同一时间只能进行一个建造或升级读条。飞船可在读条期间移动，但必须始终处于目标 1200 码操作范围内。能量在读条完成时扣除；按 `Esc`、飞船死亡、离开操作范围、建造位置变为非法或升级目标失效时，读条立即取消且不扣能量。读条期间按空格进入暂停时，读条进度会冻结，继续游戏后恢复。

### 5.7 升级

- **纯数值升级**（Lv1 → Lv2 → Lv3 ...）
- 局外养成到特定等级解锁额外能力

### 5.8 塔的携带

- 局外预先选择本次可携带的塔类型（从已解锁池中选）
- 进入战斗后只能建造预先选定的塔
- 关卡模式中：部分塔被锁定（不可选）/ 部分强制携带（新手引导）

## 六、地图系统

### 6.1 结构
- 俯视 2D
- **固定路径**（通道式），敌人不会偏离
- **超大** — 飞船移动覆盖不同区域需要时间
- 小地图仅作为**局部视角**下的战术辅助自动显示；切换到全局视角时自动隐藏，玩家不能单独开关小地图
- 路径塔只能**贴墙放置**；阻挡塔与无人机制造核心可在合法空地自由放置（均无预设槽位）

### 6.2 目标点（反应炉）

#### 主反应炉

| 等级 | HP | 产能 | 升级费 | 回本 |
|:--:|:--:|:--:|:--:|:--:|
| Lv1 | 200 | 5/秒 | — | — |
| Lv2 | 200 | 7/秒 | 100 | 50s |
| Lv3 | 200 | 9/秒 | 100 | 50s |

数量 1，不可建造、出售、移动或维修。被敌人摧毁 = 游戏失败。玩家驾驶飞船进入操作范围后点击主反应炉，可查看当前耐久与产能并通过升级读条支付能源升级；升级完成后生效，且不会回复已损失的耐久。

#### 小反应炉

| 等级 | HP | 产能 | 建造/升级费 | 回本 | 危险 |
|:--:|:--:|:--:|:--:|:--:|:--:|
| Lv1 | 300 | 2/秒 | 200 | 100s | 3 |
| Lv2 | 300 | 3/秒 | 80 | 80s | 3 |
| Lv3 | 300 | 4/秒 | 80 | 80s | 3 |

上限 5 座，不占局外携带塔位。玩家可在飞船操作范围内的合法空地建造，不能与墙体、防御建筑或出生点重叠；任意两个存活反应炉的中心点直线距离必须不小于 1000 码（距离小于 1000 码时禁止建造），主反应炉同样参与该限制。反应炉本身不作为物理阻挡。点击小反应炉可查看耐久与产能，并通过统一读条完成升级。

兜底范围为全图，对全体敌人生效；当敌人没有被无人机强制锁定、且不存在可拦截自己的合格阻挡塔时，与主反应炉共同进入反应炉候选，并按直线距离选择最近的存活反应炉。敌人一旦锁定存活反应炉，不会因新反应炉出现或距离变化切换目标；当前目标被摧毁后才重新选择最近目标。

小反应炉不可维修、出售或移动。被摧毁不影响胜负、不返还能量，但其产能立即消失；所有以它为目标的敌人立即重新执行反应炉兜底选择。该限制使小反应炉既是长期经济投资，也是会分流敌人但需要保护的次要目标。

#### 反应炉交互与反馈

- 建造栏固定提供“小反应炉”，快捷键为 `R`，不受本局携带塔型限制
- 建造小反应炉时显示每座存活反应炉的 1000 码禁建范围；预览进入禁建范围后显示为不可建造
- HUD 持续显示当前总产能与存活小反应炉数量
- 主/小反应炉均显示独立耐久条；飞船在操作范围内点击后显示明显选中环
- 反应炉面板显示当前耐久、产能、全图兜底规则和下一等级产能变化
- 建造、升级或摧毁后，总产能从该帧起更新；没有延迟结算或储能缓冲

### 6.3 路径与出生点
- 敌人出生点位于地图外围
- 多条独立通道向心汇聚至反应炉
- 波间敌人出生方向有**视觉提示**（箭头/高亮）
- 具体走哪条通道 → 玩家根据通道方向自行判断

### 6.4 关卡地图数据结构（Demo / 地图编辑器 JSON）

地图数据必须区分“碰撞墙体”和“路线设计元数据”：

- `walls` 是唯一用于 Demo 碰撞、寻路阻挡、贴墙建造判定的数据，格式为矩形列表 `[x,y,w,h]`
- `paths` 只表示设计预期路线、测试参考路线或编辑器辅助线，不参与碰撞，不等于墙体，也不能被 Demo 直接当作墙体放置
- 地图编辑器可以根据 `paths` 辅助生成通道外侧的 `walls`，但导入 Demo 时仍以最终显式导出的 `walls` 为准
- `routes` 为旧版编辑器字段别名，仅用于兼容导入；新数据与 GDD 统一使用 `paths`
- `unitScale` 表示“游戏距离单位 → 屏幕像素”的缩放比例。调小该值等于在同一画布上制作更大的物理地图；射程、速度、反应炉 1000 码限制、飞船操作半径、建筑显示尺寸等需要按该比例统一显示
- `worldSize` 记录地图逻辑画布尺寸。当前 Demo 导入地图时要求 `worldSize` 与 Demo 运行画布一致，否则使用内置地图，避免坐标错位

规范 JSON 示例：

```json
{
  "schemaVersion": 1,
  "kind": "level-map",
  "id": "level1",
  "name": "边境蛇道",
  "unitScale": 0.82,
  "worldSize": { "w": 7356, "h": 4144 },
  "walls": [[0, 0, 7356, 80]],
  "spawns": [[260, 420]],
  "reactor": { "x": 6900, "y": 3720 },
  "paths": [[[260, 420], [760, 420], [6900, 3720]]],
  "pockets": [],
  "notes": []
}
```

| 字段 | 类型 | 必需 | 说明 |
|---|---|:--:|---|
| `schemaVersion` | number | 是 | 当前为 `1`，用于后续兼容升级 |
| `kind` | string | 是 | `level-map` / `endless-map` / `editor-map` |
| `id` | string | 是 | 地图或关卡唯一标识 |
| `name` | string | 是 | 地图显示名称 |
| `unitScale` | number | 是 | 距离单位缩放比例 |
| `worldSize` | object | 是 | `{w,h}`，地图逻辑画布尺寸 |
| `walls` | array | 是 | 矩形墙体列表，唯一碰撞源 |
| `spawns` | array | 是 | 出生点列表，`spawns[i]` 对应第 `i` 条入口 |
| `reactor` | object | 是 | 主反应炉坐标 `{x,y}` |
| `paths` | array | 否 | 设计路线元数据，每条路线由点列表组成 |
| `pockets` | array | 否 | 可建造空地、战术口袋或编辑器标注，不直接改变碰撞 |
| `notes` | array | 否 | 仅供设计查阅，不在可玩地图中显示 |

## 八、波次与准备阶段

### 8.1 准备阶段
- 每波结束后进入准备阶段
- **固定时长**，倒计时结束自动开始下一波
- 飞船可自由移动、建造、升级
- 敌人出生方向有提示
- 关卡模式和无尽模式准备时长可不同

### 8.2 波次
- 所有我方状态**跨波次保留**：塔、无人机数量、能源、飞船等级均不重置
- 关卡模式使用独立的固定波次配置；无尽模式使用隐藏的“波次威胁预算”生成阵容，预算数值不向玩家显示

#### 8.2.1 无尽模式威胁预算

- 第 N 波基础预算：`5 + 3 × (N-1)`，即第1/5/10/20波分别为5/17/32/62点
- 系统只从当前波次已经解锁的敌人中组建阵容；选择一只敌人会扣除其威胁成本，剩余预算无法购买其他敌人时用 E1 补足
- 威胁成本只控制波次构成，不改变敌人的危险等级、击杀回能或属性
- E5、E8、E10归为“特殊单位”，每波消耗的威胁预算合计不得超过该波总预算的35%；同一种特殊单位每波最多出现2只
- 每波至少35%的预算用于E1、E2、E7等直接进攻单位，避免整波均由肉盾或召唤单位组成
- 在满足上述约束后随机选择敌人并打乱出生顺序；多出生方向时尽量轮流分配，避免全部高威胁单位集中在同一路线

| 敌人 | E1 | E2 | E3 | E4 | E5 | E6 | E7 | E8 | E9 | E10 | E11 | E12 |
|---|:--:|:--:|:--:|:--:|:--:|:--:|:--:|:--:|:--:|:--:|:--:|:--:|
| 威胁成本 | 1 | 1 | 3 | 3 | 7 | 6 | 2 | 3 | 5 | 7 | 2 | 5 |

| 首次解锁波次 | 1 | 2 | 3 | 4 | 5 | 6 | 7 | 8 | 9 |
|---|---|---|---|---|---|---|---|---|---|
| 新增敌人 | E1 | E2 | E3 | E4、E7 | E8 | E5、E6 | E11 | E9、E10 | E12 |

定位：关卡模式用于沉淀经过地图编辑器验证的固定地图与固定 10 波配置。关卡地图保持干净，只显示墙体、入口与主反应炉；不在可玩地图中绘制“前线经济点”、建议圈、突袭线圈或文字说明。

|---:|---|---:|---:|---|

#### 8.2.3 无尽地图：无尽模式一

定位：大物理地图白盒，用于验证 1000 码反应炉间距限制是否能迫使玩家扩大战线，而不是把多座小反应炉和主防线堆在同一路。

- 地图名：**无尽模式一**
- 地图结构：8 个入口，左上/左中/左下、右上/右中/右下、上中与下中多方向进攻；主反应炉位于地图中央，路线以封闭廊道和多段折线形成长距离推进压力
- 地图尺度：视觉画布仍使用基础尺寸 7356×4144；本图使用 `unitScale=0.48`，即 1 个游戏距离单位折算为 0.48 像素。所有射程、速度、弹道距离、反应炉 1000 码间距、飞船操作半径，以及反应炉自身显示/碰撞尺寸均按该比例显示，因此同一画布约等于 `1 / 0.48 ≈ 2.08` 倍物理距离。

- 防守目标：玩家可以选择先守中央、先扩单侧外线、或用无人机/飞船在远端救火；不同选择会改变飞船奔波距离和能量收益节奏
- 寻路检查：当前白盒 8 个入口均可抵达主反应炉；测试路径长度约为设计路线的 0.96~1.08 倍，未发现明显抄近路问题

### 8.3 波次难度递增
- 第 1 波：敌人 HP = 基础值 × 1.0
- 第 N 波：敌人 HP = 基础值 × [1.0 + (N-1) × 0.1]
- 示例：第 5 波 1.4×，第 10 波 1.9×，第 20 波 2.9×
- **仅 HP 随波次成长**；伤害、攻击频率、移动速度均保持基础值
- E2 加速态速度同样固定，只受减速和 E8 光环影响
- **递增仅影响敌人，不影响塔和我方单位**

### 波次难度递增
- 第 1 波：敌人 HP = 基础值 × 1.0
- 第 N 波：敌人 HP = 基础值 × [1.0 + (N-1) × 0.1]
- 仅 HP 随波次成长；伤害、攻击频率、移动速度均保持基础值
- E2 加速态速度同样固定，只受减速和 E8 光环影响
- 递增仅影响敌人，不影响塔和无人机
