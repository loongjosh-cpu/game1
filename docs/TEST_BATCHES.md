# 游戏逻辑大检查：分批测试计划

目标：把“大而全”的测试拆成可独立执行、可独立提交的小批次。每一批先做自动化/静态验证，再做少量人工实机验证，避免测试范围发散。

## 前置准备

- 固定入口：
  - Demo：`demo.html`
  - 地图编辑器：`map-editor.html`
  - 预检脚本：`node tools/verify-game-preflight.js`
  - 地图编辑器预检：`node tools/verify-map-editor.js`
- 每批测试完成后记录：
  - 测试范围
  - 自动化结果
  - 人工验证场景
  - 发现的问题
  - 已修复/待修复

## 批次 0：预检与数据完整性

目的：确认项目基础数据、脚本顺序、GDD 关键口径没有明显断裂。

覆盖：

- `demo.html` 引用脚本是否存在、语法是否通过。
- 防御塔、敌人、芯片、关卡、地图数据的基本完整性。
- 芯片引用的塔是否存在，互斥组/费用是否有效。
- 关键 GDD 口径与代码同步检查，例如 B3 重型炮塔等级数。
- 地图编辑器能否加载关卡列表。

主要命令：

```bash
node tools/verify-game-preflight.js
node tools/verify-map-editor.js
```

## 批次 1：数据表与 GDD 对齐

目的：检查所有塔、怪物、反应炉、芯片、波次规则是否与 GDD 一致。

覆盖：

- 路径塔 P1-P7：伤害、间隔、范围、目标数、费用、升级费、特殊规则。
- 阻挡塔 B1-B7：HP、危险等级、范围、攻击/治疗/自爆/残骸规则。
- 无人机核心 D1-D3：核心 HP、危险等级、无人机上限、无人机 HP、产速、补员能耗、攻击/治疗逻辑。
- 主/小反应炉：HP、产能、升级、距离限制、不可维修/出售。
- 敌人 E1-E14：HP、速度、伤害、攻速、危险等级、特殊机制。
- 芯片：售价、互斥、装备目标、效果口径。

输出：

- 一张“代码 vs GDD”差异表。
- 差异按“必须修 / 需要设计确认 / 可延后表现层”分类。

## 批次 2：建造、升级、出售、读条与放置规则

目的：验证玩家基础操作不会出现无响应、非法放置、费用扣错、读条异常。

覆盖：

- 路径塔贴墙吸附、阻挡塔自由放置、无人机核心放置。
- 塔体重叠、墙体碰撞、地图边界限制。
- 建造读条、升级读条、取消/暂停时行为。
- 升级按钮、出售按钮、出售返还芯片效果。
- 小反应炉 1000 码距离限制。

人工场景：

- 在每种地图上尝试合法/非法放置。
- 建造中暂停、ESC、返回主页。
- 连续升级/出售，确认能量与 UI 同步。

## 批次 3：敌人寻路、嘲讽与目标选择

目的：重点检查之前反复出问题的“目标选择”和“离开范围后嘲讽失效”。

覆盖：

- 敌人沿路线移动，不走墙、不从非法捷径穿过。
- 多个阻挡塔/无人机核心/反应炉同时存在时的锁定优先级。
- 敌人离开塔作用范围后不继续被嘲讽。
- 更高危险塔出现时是否按规则切换。
- 反应炉只作为兜底目标，不压制合格阻挡塔。
- 远程怪 E11/E12 是否遵守路径与拦截规则。

人工场景：

- B1/B3/B4/B5/B6/B7 混合摆放。
- 装备 B3 挑衅炮膛、B1 低热残骸芯片后观察目标选择。
- 小反应炉多点分布，观察漏怪兜底目标。

## 批次 4：塔攻击、治疗、DOT、弹道与特效

目的：验证每座塔的核心战斗逻辑，尤其是 AOE、弹道到点结算、毒/冰冻/减速。

覆盖：

- P1-P7 全等级攻击逻辑。
- B1-B7 全等级攻击/治疗/残骸/自爆/脉冲。
- P2/P3/B4 弹道或电磁表现与结算时机。
- P4 减速/冰冻芯片。
- P6/B6/P7 标准中毒、毒药芯片、毒爆结算。
- B5 治疗、过量护盾、无人机治疗芯片。

人工场景：

- 单怪、密集怪、快速怪、高危险怪分别测试。
- 每种塔至少测试 Lv1 与满级。
- 芯片装备前后对比。

## 批次 5：无人机系统

目的：单独检查无人机，因为它涉及移动、目标选择、补员、回血、核心被攻击、墙体阻挡。

覆盖：

- D1/D2/D3 生产、上限、补员耗能、自动补员开关。
- 无人机不穿墙。
- 巡逻范围受核心影响，追击可离开核心范围。
- 已进入战斗的无人机不频繁换目标导致脱战。
- 多无人机分配目标，避免过量集中。
- D2 死亡复活芯片、D1 首伤减免/死亡爆炸芯片。
- D3 维修、返航、维修消耗自身 HP、不可战斗。

人工场景：

- 三个核心同时建造，观察是否有无人机卡住。
- 高速怪穿线，观察无人机追击与回航。
- 大量敌人压线，观察目标分配。

## 批次 6：波次、经济、奖励与模式结算

目的：确认关卡模式与无尽模式的出敌、经济、奖励不会互相污染。

覆盖：

- 关卡模式固定波次。
- 第 5 关后 15 波配置。
- 无尽模式随机预算、敌人解锁、HP 成长。
- 击杀回能 = 威胁成本 × 3。
- 主反应炉/小反应炉产能。
- 主反应炉芯片前 5 波每波 120 能量。
- 无尽主动退出、失败、撑过 5 波的星核结算。
- 关卡通关星核与解锁逻辑。

人工场景：

- 关卡失败后重玩，确认出敌记忆稳定。
- 无尽主动退出，确认星核按已完成波数结算。
- 多小反应炉经济压力测试。

## 批次 7：局外系统、商店、整备舱与存档

目的：验证局外养成不会出现购买、装备、卸下、互斥、存档恢复错误。

覆盖：

- 专精商店购买、星核扣除、已购买状态。
- 整备舱拖动装备、点击替代装备、卸下。
- 同塔芯片互斥、毒药芯片三选一。
- 未拥有芯片不可装备。
- 装备后进入战斗生效，返回主页后仍保存。
- localStorage 存档兼容。

人工场景：

- 每个互斥组至少切换一次。
- 清空/模拟旧存档后启动。

## 批次 8：UI、视角、暂停、地图与编辑器回归

目的：检查玩家路径完整性和之前高频 UI bug。

覆盖：

- 主页面层级、关卡模式、无尽模式、档案、商店、整备舱。
- 视角切换：全局视角/局部视角。
- 小地图只在局部视角显示。
- ESC/空格暂停逻辑，能退出时优先退出，不能退出时暂停。
- 档案：塔/怪物图标、介绍、费用、危险等级、滚动条。
- 地图编辑器：关卡列表、拖动墙体/出生点/反应炉、导入导出。

人工场景：

- 从主页面进入每个二级页面再返回。
- 局内暂停页切视角，再回战斗。
- 地图编辑器刷新后确认所有关卡仍显示。

## 批次 9：全局回归与压力测试

目的：在前面批次都通过后做最终串联。

覆盖：

- 关卡模式至少跑 2 张地图。
- 无尽模式至少跑到第 10 波。
- 典型芯片组合：
  - 毒系组合
  - 阻挡塔组合
  - 无人机组合
  - 经济组合
- 长时间运行观察性能、卡死、内存、UI 同步。

输出：

- 最终 bug 清单。
- 下一轮优先级：致命 / 高 / 中 / 低。
